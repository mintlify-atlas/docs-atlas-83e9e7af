---
title: 'Get current user'
description: "Verify a session token and retrieve the authenticated user's email and role."
---

Verifies the JWT passed in the `token` request header and returns the authenticated user's email address and role. Use this endpoint to confirm a valid session and determine what the current user is permitted to do.

<Note>
  The token is read from the `token` request header, not from the `Authorization` header and not from the cookie. Make sure your client sets the header explicitly.
</Note>

## Request

<ParamField header="token" type="string" required>
  The JWT obtained from the [login](/api/auth/login) response body.
</ParamField>

No request body is required.

## Response

<ResponseField name="email" type="string">
  The authenticated user's email address. Only present when `okk` is `true`.
</ResponseField>

<ResponseField name="role" type="string">
  The user's role: `member`, `librarian`, or `admin`. Only present when `okk` is `true`.
</ResponseField>

<ResponseField name="okk" type="boolean" required>
  `true` when the token is valid and the user was found; `false` otherwise.
</ResponseField>

<ResponseField name="message" type="string">
  Error description. Present only when `okk` is `false`.
</ResponseField>

## Code examples

<CodeGroup>

```bash curl
curl --request POST \
  --url https://api.libraease.com/email \
  --header 'token: eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...'
```

```javascript fetch
const token = localStorage.getItem('token');

const response = await fetch('https://api.libraease.com/email', {
  method: 'POST',
  headers: {
    token: token,
  },
});

const data = await response.json();

if (data.okk) {
  console.log(data.email); // "jsmith@example.com"
  console.log(data.role);  // "member" | "librarian" | "admin"
}
```

</CodeGroup>

## Response examples

<CodeGroup>

```json 200 Success
{
  "email": "jsmith@example.com",
  "role": "member",
  "okk": true
}
```

```json 200 User not found
{
  "message": "no email found",
  "okk": false
}
```

```json 401 No token provided
{
  "message": "no token",
  "okk": false
}
```

```json 400 Server error
{
  "message": "error",
  "okk": false
}
```

</CodeGroup>

<Warning>
  The `401` response is the only case where the HTTP status code differs from `200` for non-server errors. All other failure responses return `200` — check `okk` in every case.
</Warning>
