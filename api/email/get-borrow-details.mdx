---
title: 'Send borrow details email'
description: 'Send a borrowing summary email to a library member listing the books they have borrowed.'
---

<Note>
  This endpoint does **not** require an authentication token. It is mounted before the token-verification middleware in `index.js` under the `/api` prefix.
</Note>

`POST /api/getBorrowDetails`

Sends an HTML summary email to the specified address using Nodemailer and Mailgen. The email includes a table of the member's borrowed books with title, author, ISBN, and due date. Members are reminded to return books by the due dates and can click through to their account.

<Warning>
  This endpoint sends a real email. Ensure `EMAIL` and `PASSWORD` are correctly configured in `backend/.env` before calling this endpoint. See [environment variables](/configuration/environment-variables) for setup instructions.
</Warning>

## Body

<ParamField body="userEmail" type="string" required>
  The email address of the library member to send the borrow summary to.
</ParamField>

## Response

<ResponseField name="msg" type="string">
  Confirmation message: `"you should receive an email"`.
</ResponseField>

### Error responses

| Status | Description |
| ------ | ----------- |
| `201`  | Email sent successfully. |
| `500`  | Failed to send email — check Nodemailer configuration. |

<CodeGroup>

```bash cURL
curl --request POST \
  --url http://localhost:5000/api/getBorrowDetails \
  --header 'Content-Type: application/json' \
  --data '{
  "userEmail": "member@example.com"
}'
```

```javascript JavaScript
const response = await fetch('http://localhost:5000/api/getBorrowDetails', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    userEmail: 'member@example.com',
  }),
});

const data = await response.json();
```

</CodeGroup>

```json 201 — success
{
  "msg": "you should receive an email"
}
```

```json 500 — email send failure

  "error": { ... }
```
