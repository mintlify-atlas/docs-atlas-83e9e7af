---
title: 'Get borrow details'
description: 'Retrieve the full borrow history and item-level details for a user.'
---

<Note>
  This is a utility endpoint. It does **not** require an authentication token
  because it is mounted before the token-verification middleware in `index.js`.
</Note>

`POST /api/getBorrowDetails`

Returns the borrow history for a user, including all checked-out items with their ISBNs, quantities, checkout dates, return dates, return status, and any outstanding fines. This endpoint is used to populate detailed borrowing-summary emails.

## Body

<ParamField body="userId" type="string" required>
  ObjectId of the user whose borrow history you want to retrieve.
</ParamField>

## Response

<ResponseField name="borrowDetails" type="object[]">
  Array of checkout records belonging to the user.

  <Expandable title="borrow record properties">
    <ResponseField name="_id" type="string">
      Unique MongoDB identifier for the checkout record.
    </ResponseField>

    <ResponseField name="items" type="object[]">
      All book items included in this checkout.

      <Expandable title="item properties">
        <ResponseField name="isbn" type="string" required>
          ISBN of the borrowed book.
        </ResponseField>

        <ResponseField name="quantity" type="number" required>
          Number of copies borrowed.
        </ResponseField>

        <ResponseField name="checkoutDate" type="string" required>
          ISO 8601 date-time string for when the checkout occurred.
        </ResponseField>

        <ResponseField name="returnDate" type="string" required>
          Return due date in `YYYY-MM-DD` format.
        </ResponseField>

        <ResponseField name="returned" type="boolean">
          Whether this item has been returned.
        </ResponseField>

        <ResponseField name="fines" type="number">
          Outstanding fine amount for this item. `0` if none.
        </ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

### Error responses

| Status | Description |
| ------ | ----------- |
| `400`  | `userId` is missing or not a valid ObjectId. |
| `404`  | No borrow records found for the given user. |
| `500`  | Unexpected server error. |

<CodeGroup>

```bash cURL
curl --request POST \
  --url https://api.libraease.com/api/getBorrowDetails \
  --header 'Content-Type: application/json' \
  --data '{
  "userId": "663f0a1b2c3d4e5f6a7b8c9d"
}'
```

```javascript JavaScript
const response = await fetch('https://api.libraease.com/api/getBorrowDetails', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    userId: '663f0a1b2c3d4e5f6a7b8c9d',
  }),
});

const data = await response.json();
```
