---
title: 'Quickstart'
description: 'Get LibraEase running locally in under ten minutes. You will start both the Express backend and the Vite frontend, then create your first librarian and member accounts.'
---

<Note>
  LibraEase runs as two separate dev servers. The backend runs on the port you configure in `.env`; the frontend runs on port **5173** by default (Vite). Both must be running at the same time for the app to work.
</Note>

<Steps>

  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/your-org/libraease.git
    cd libraease
    ```
  </Step>

  <Step title="Install backend dependencies">
    Navigate to the backend directory and install all required packages.

    <CodeGroup>

    ```bash npm
    cd backend
    npm install
    npm install nodemailer mailgen
    ```

    ```bash yarn
    cd backend
    yarn
    yarn add nodemailer mailgen
    ```

    </CodeGroup>

    This installs express, mongoose, bcrypt, jsonwebtoken, nodemailer, mailgen, dotenv, cors, cookie-parser, body-parser, and nodemon.
  </Step>

  <Step title="Configure backend environment variables">
    Create a `.env` file in the `backend/` directory:

    ```bash
    cp .env.example .env
    ```

    Then populate every variable:

    ```ini backend/.env
    # Server
    PORT=5000
    baseUrl=http://localhost

    # Authentication
    SECRET_KEY=your_jwt_secret_key_here
    PASSWORD_KEY=10

    # Email (nodemailer + mailgen)
    EMAIL=your_gmail_address@gmail.com
    PASSWORD=your_gmail_app_password

    # Database
    DB_URL=mongodb://localhost:27017
    DB_NAME=libraease
    ```

    <Warning>
      `SECRET_KEY` and `PASSWORD_KEY` must be long, random strings. Do not commit `.env` to version control.
    </Warning>

    <Tip>
      For `EMAIL` and `PASSWORD`, use a Gmail account with an App Password rather than your main account password. App Passwords are required when 2-Step Verification is enabled on your Google account.
    </Tip>
  </Step>

  <Step title="Start the backend server">
    <CodeGroup>

    ```bash npm
    npm run dev
    ```

    ```bash yarn
    yarn dev
    ```

    </CodeGroup>

    You should see:

    ```
    Server is Running on http://localhost at 5000
    ```

    The backend is now listening at `http://localhost:5000`.
  </Step>

  <Step title="Install frontend dependencies">
    Open a new terminal, then navigate to the frontend directory:

    <CodeGroup>

    ```bash npm
    cd frontend
    npm install
    ```

    ```bash yarn
    cd frontend
    yarn
    ```

    </CodeGroup>
  </Step>

  <Step title="Configure frontend environment variables">
    Create a `.env` file in the `frontend/` directory:

    ```ini frontend/.env
    # Base URL of the backend API — no trailing slash
    VITE_API_URL=http://localhost:5000
    ```

    Vite exposes only variables prefixed with `VITE_` to the browser bundle. Any variable without this prefix will be silently ignored at runtime.
  </Step>

  <Step title="Start the frontend">
    <CodeGroup>

    ```bash npm
    npm run dev
    ```

    ```bash yarn
    yarn dev
    ```

    </CodeGroup>

    Vite will print:

    ```
    VITE v4.x.x  ready in Xms

    ➜  Local:   http://localhost:5173/
    ```

    Open `http://localhost:5173` in your browser.
  </Step>

  <Step title="Sign up as a librarian and add a book">
    1. On the signup page, fill in your details and select the **Librarian** role.
    2. Submit the form — this calls `POST /signup` and creates your account.
    3. Log in with your new credentials (`POST /login`).
    4. From the librarian dashboard, click **Add Book** and fill in the book details.
    5. Submit — this calls `POST /book/addbook` using the JWT issued at login.

    <Tip>
      The librarian role gives you full catalogue access: add, update, and delete books, and view all borrower activity via `POST /book/getallbooks`.
    </Tip>
  </Step>

  <Step title="Sign up as a member and check out a book">
    1. Open a private/incognito window (or log out) and navigate to `http://localhost:5173`.
    2. Sign up again, this time selecting the **Member** role.
    3. Log in as the member.
    4. Browse available books (`POST /book/getbooks`) and select a title.
    5. Click **Checkout** — this calls `POST /book/checkout` and records the loan against your account.

    <Note>
      Due-date notifications are sent by email via `POST /api/getDueDate`. Make sure your backend `EMAIL` and `PASSWORD` environment variables are configured correctly before testing email features.
    </Note>
  </Step>

</Steps>
