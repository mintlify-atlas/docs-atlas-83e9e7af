LibraEase sends a transactional email to the member immediately after a successful checkout. Emails are delivered through Gmail using Nodemailer and formatted with Mailgen.

## What triggers an email

An email is sent automatically when the `checkout` controller successfully saves a new checkout record. No manual action is required — the email fires as part of the same request that creates the record.

## Email content

The confirmation email contains a table listing every book included in the checkout:

| Column | Description |
|---|---|
| Title | Full book title |
| Author | Author name |
| ISBN | Book identifier |
| Quantity | Number of copies borrowed |
| Days | Number of days until the book is due |

The email is generated by Mailgen using the `"default"` theme and is branded with the LibraEase product name.

## Gmail configuration

<Warning>
  Do not use your regular Gmail password. Gmail requires an **App Password** for programmatic SMTP access. Using your account password will cause authentication to fail and may trigger a security lockout.
</Warning>

To generate an App Password:

1. Go to your Google Account → **Security**.
2. Enable **2-Step Verification** if it is not already on.
3. Under **2-Step Verification**, select **App passwords**.
4. Create a new app password and copy the 16-character code.

Add the credentials to your `.env` file:

```bash .env
EMAIL=your-address@gmail.com
PASSWORD=abcd efgh ijkl mnop
SECRET_KEY=your-jwt-secret
```

## How the mailer is configured

```js
const nodemailer = require('nodemailer');
const Mailgen = require('mailgen');

const config = {
  service: 'gmail',
  auth: {
    user: process.env.EMAIL,
    pass: process.env.PASSWORD,
  },
};

const transporter = nodemailer.createTransport(config);

const MailGenerator = new Mailgen({
  theme: 'default',
  product: {
    name: 'LibraEase',
    link: 'https://google.com',
  },
});
```

The `transporter` is called inside the `checkout` controller after the checkout document is saved successfully. If the email send fails, the checkout record is already persisted — the email failure does not roll back the transaction.

## Email utility endpoints

Two dedicated endpoints under `/api` can be called to trigger targeted emails to a specific member:

<Tip>
  Both endpoints accept a `userEmail` body parameter and send a formatted HTML email. They do not return data — they trigger email delivery.
</Tip>

```http
POST /api/getDueDate
```

Sends a **due-date reminder email** to the specified member. The email lists their checked-out books and warns that a charge of 10 Rs per day applies for overdue returns.

```http
POST /api/getBorrowDetails
```

Sends a **borrow summary email** to the specified member. The email includes a table of all borrowed books with title, author, ISBN, and due date.

## Troubleshooting

| Issue | Likely cause | Fix |
|---|---|---|
| `Invalid login` SMTP error | Wrong password or App Password not enabled | Generate a fresh App Password and update `.env` |
| Email not received | Spam filter or incorrect `EMAIL` value | Check spam folder; confirm `EMAIL` in `.env` matches the Gmail account |
| `Less secure app access` error | Google has deprecated this setting | Switch to App Passwords (see above) |
