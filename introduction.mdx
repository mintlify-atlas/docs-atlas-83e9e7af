# LibraEase

LibraEase is a full-stack library management system that makes it easy to catalog books, manage member checkouts, and send automated email notifications. It provides role-based access so librarians can manage the collection while members can browse and borrow books.

## Key features

<CardGroup cols={2}>
  <Card title="Role-based access" icon="users">
    Separate workflows for librarians and members, with protected routes enforced on both frontend and backend.
  </Card>
  <Card title="Book catalog" icon="books">
    Full CRUD for books — add new titles, update metadata, remove outdated entries, and search the full catalog.
  </Card>
  <Card title="Checkout & returns" icon="calendar-check">
    Members select books, choose a return date, and the system records the checkout with automatic due-date tracking.
  </Card>
  <Card title="Email notifications" icon="envelope">

  </Card>
  <Card title="JWT authentication" icon="lock">

  </Card>

## Architecture overview

LibraEase is split into two independent applications:

| Layer | Technology |
|---|---|
| Frontend | React 18 + Vite, Material UI, React Router v6 |
| Backend | Node.js + Express, JWT auth, Nodemailer |
| Database | MongoDB via Mongoose ODM |

The backend exposes a REST API that the React frontend consumes. Authentication uses JWT tokens passed via HTTP-only cookies or request headers.

## User roles

LibraEase supports three roles:

- **Librarian** — can add, update, and delete books; view all checkouts
- **Member** — can browse the full catalog, check out books, and view their own borrow history
- **Admin** — reserved for future administrative features

<Note>
  Role assignment happens at signup. Each user selects either `member` or `librarian` during registration.
</Note>
